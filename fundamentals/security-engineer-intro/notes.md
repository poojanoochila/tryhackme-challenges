# Notes: Security Engineer Intro

## Who is a Security Engineer?
A Security Engineer is responsible for designing, implementing, and maintaining secure systems to protect against cyber threats.

## Core Responsibilities
- Implement security controls (firewalls, IDS/IPS)
- Monitor systems for suspicious activity
- Identify vulnerabilities and apply fixes
- Support incident response activities

## Security Engineer vs SOC Analyst

Security Engineer:
- Focus: Building and securing infrastructure
- Works on prevention and system design

SOC Analyst:
- Focus: Monitoring and responding to threats
- Works on detection and incident handling

## Key Security Functions

### 1. Prevention
- Hardening systems
- Configuring firewalls
- Applying patches

### 2. Detection
- Monitoring logs
- Identifying anomalies
- Generating alerts

### 3. Response
- Investigating incidents
- Containing threats
- Reporting findings

## Importance of Logging
Logs are critical for:
- Detecting attacks
- Investigating incidents
- Understanding attacker behavior

Common log sources:
- Authentication logs
- Network traffic logs
- Application logs
